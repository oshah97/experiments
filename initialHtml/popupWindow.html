The "popup" method puts the call-counter table inside of a popup that opens within the window. At this time, the table inside the popup can scorll up and down but cannot scroll in the horizontal direction. The main issue with the table was the weird formatting when the window was sized too small, but that problem persits within the popup method.

Update: popup window automatically resizes the size of the popup when window is resized, and does scroll horiontally when there is overflow. Formatting of table still funky.

Update: Popup window is "draggable"! jQuery UI was added as source script and that allowed for the creation of a function that made the popup draggable, within the window. Also resizable, but that might be useless. When it is resizable, it cancels the auto-resizing of the popup as the window is resized.
